# cpk-ui 0.7.0

cpk-ui 0.7.0 is a release focused on predictable component behavior across
iOS, Android, and web. It pairs a stricter styling contract with broader state
coverage and a Storybook workbench designed for repeated component review.

## Install

```bash
bun add cpk-ui@0.7.0 kstyled@0.4.1
bun add --dev babel-plugin-kstyled@0.4.1
```

Configure the compiler before Reanimated:

```js
plugins: [
  ['babel-plugin-kstyled', {strict: true}],
  'react-native-reanimated/plugin',
];
```

## Highlights

- All 21 public components have automated tests and documented Storybook
  states.
- Inputs and disclosure controls demonstrate real controlled interactions.
- Dialog, snackbar, loading, selection, and floating-action states expose
  clearer accessibility semantics.
- Storybook provides global themes, responsive viewports, source, controls,
  accessibility checks, and a structured component taxonomy.
- The icon runtime no longer imports or publishes the 6.4 MB IcoMoon selection
  file.

## Migration

1. Upgrade cpk-ui, kstyled, and the Babel plugin together.
2. Enable kstyled strict mode and resolve any compiler diagnostics.
3. Verify custom themes in both light and dark modes.
4. Recheck controlled inputs, modal refs, and custom press handlers.
5. Run representative screens on iOS, Android, and web before rolling out.

No component was intentionally removed. The required kstyled 0.4 contract is
the main upgrade boundary.
